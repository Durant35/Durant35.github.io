Autoware

From: http://www.pdsl.jp/fot/autoware/

Github: https://github.com/CPFL/Autoware

​ 自动运行软件，Autoware。

​ Autoware是一种基于Linux和ROS自动运行系统的开放源代码软件。通过传感器，如激光雷达，摄像头，全球导航卫星系统感知周边环境，同时确认车辆位置与周围的物体，实现根据导航系统提供的路线完成自动驾驶。

备注: 名古屋大学，长崎大学，AIST联合发布

基本功能

  • 3D Localization(三维定位)
  • 3D Mapping(三维地图生成)
  • Path Planning(路径规划)
  • Path Following(路径跟随)
  • Accel/Brake/Steering Control(油门/刹车/方向盘控制)
  • Data Logging
  • Car/Pedestrian/Object Detection(物体检测：车辆、行人、其他)
  • Traffic Signal Detection(交通标志识别)
  • Traffic Light Recognition(交通灯检测)
  • Lane Detection(车道线识别)
  • Object Tracking(移动追踪)
  • Sensor Calibration(传感器标定)
  • Sensor Fusion(多传感器融合)
  • Cloud-oriented Maps
  • Connected Automation
  • Smartphone Navigation
  • Software Simulation
  • Virtual Reality

配置

  • 爱三科技的高精度三维的地图
  • 贾瓦德的GNSS卫星定位技术
  • HOKUYO，Velodyne，激光雷达
  • 英特尔处理器，运行Linux
  • NVIDIA GPGPU：GeForce、Tegra
  • …

Demo(户外测试)

3D Mapping

  • 爱三科技提供的用于自动驾驶的三维高精度地图。
  • 另外，使用SLAM技术来实时生成三维地图。

自主定位

  • 利用实时的Lidar点云数据和加载的高精度三维地图，通过基于Normal Distributions Transform (NDT)正态分布变换的点云匹配算法能够实现实时定位。定位误差在10cm以内。
  • 实时定位信息还可以用于更新加载的高精度三维地图。

交通灯识别

​ 结合自主定位的结果推算出高精度三维地图中交通灯的结果，以准确地计算出交通灯在摄像头采集到的图像中的投影，进而通过图像处理进行颜色鉴别对交通灯信号进行检测。

Object Detection

​ 通过摄像头采集实时图像，使用Deformable Part Models(DPM) 图像识别算法可以检测到汽车或者行人；使用卡尔曼滤波器进行跟踪能够提高检测进度。此外，使用三维激光雷达进行数据融合还能够得到距离被检测对象的距离。

Path Planning

  • 并行车道线内的路径规划

导航系统给出路由到目标点的一条车道路线，车道线内的路径规划系统结合车道线内汽车的运动学轨迹生成局部路径，并在生成的轨迹中选择最安全的路线行驶。

  • 基于预描点的路径规划

在生成的路径上每隔1m做一个标记(航点) ，通过依次遍历相邻的每个航点输出对应的控制；在曲线上航点通常每隔两个或更远的航点去做规划，以实现较为稳定的自主控制。

系统集成

​ 集成的系统包括：三维地图上的自主定位、物体检测、路径生成、路径追踪、可视化工具。

用户界面

​ Autoware的开发人员通过使用Runtime Manager，能够完成定位、物体检测、路径追踪等简单的处理。

  • Runtime Manager NDT(应该是Navigation Development Technique)可以加载高精度地图、进行点云数据匹配实现定位、构建三维地图等。
  • Autoware的用户可以使用开发的平板UI,利用导航和路径的生成，完成自动驾驶模式转换等简单处理。
  • 利用车载显示器和Oculus投影元件可以进一步将自动驾驶系统所使用的三维地图信息可视化。
